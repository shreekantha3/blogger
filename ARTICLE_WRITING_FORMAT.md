## Content Structure

### Overall Structure

A RankMath-optimized article follows this structure:

```
<h1>Main Title (Primary Keyword)</h1>
    ↳ <h2>Introduction</h2>
        ↳ <h3>Background</h3>
        ↳ <h3>Problem Statement</h3>
    ↳ <h2>Background</h2>
        ↳ <h3>Historical Context</h3>
        ↳ <h3>Key Developments</h3>
    ↳ <h2>Current Situation</h2>
        ↳ <h3>Trend Analysis</h3>
        ↳ <h3>Market Overview</h3>
    ↳ <h2>Analysis</h2>
        ↳ <h3>Deep Dive 1</h3>
        ↳ <h3>Deep Dive 2</h3>
    ↳ <h2>Implications</h2>
        ↳ <h3>Short-term Impact</h3>
        ↳ <h3>Long-term Outlook</h3>
    ↳ <h2>Conclusion</h2>
        ↳ <h3>Key Takeaways</h3>
        ↳ <h3>Next Steps</h3>
```

### Content Flow

1. **Introduction** (H2): Hook the reader, state the problem
2. **Background** (H2): Provide context and history
3. **Current Situation** (H2): Present current state of affairs
4. **Analysis** (H2): Deep dive into data, facts, insights
5. **Implications** (H2): Discuss consequences and future outlook
6. **Conclusion** (H2): Summarize and provide actionable takeaways

## Heading Hierarchy

### H1 Tag Requirements

- **Exactly one H1 per page**
- Matches the page title
- Contains primary keyword naturally
- Under 70 characters (preferably 50-60)
- Title case formatting

**Example:**
```html
<h1>Complete Guide to Python Programming for Beginners</h1>
```

### H2 Tag Requirements

- **3-7 H2 sections** (5-7 recommended for 2,500+ word articles)
- Use semantic keywords
- Natural language flow
- Each H2 should be a complete thought

**Example:**
```html
<h2>Introduction to Python Programming</h2>
<h2>Setting Up Your Development Environment</h2>
<h2>Python Basics: Variables and Data Types</h2>
<h2>Control Flow: Conditionals and Loops</h2>
<h2>Functions and Modules</h2>
<h2>Advanced Topics</h2>
<h2>Conclusion and Next Steps</h2>
```

### H3 Tag Requirements

- **2-3 H3 subsections per H2**
- Provide detailed breakdown of H2 topic
- Use secondary keywords or LSI variations
- Maintain semantic hierarchy (no skipping levels)

**Example:**
```html
<h2>Python Basics: Variables and Data Types</h2>
<h3>Understanding Variables</h3>
<h3>Data Types in Python</h3>
<h3>Type Conversion and Casting</h3>
```

### Invalid Patterns to Avoid

❌ **Skipping Levels:**
```html
<!-- WRONG -->
<h1>Main Title</h1>
<h3>Subsection without H2</h3>
```

✅ **Correct Pattern:**
```html
<!-- CORRECT -->
<h1>Main Title</h1>
<h2>Main Section</h2>
<h3>Subsection</h3>
```

❌ **Multiple H1 Tags:**
```html
<!-- WRONG -->
<h1>First Title</h1>
<h1>Second Title</h1>
```

## Keyword Placement

### Primary Keyword Distribution

| Location | Requirement | Example |
|----------|-------------|---------|
| **H1** | 1 time | `<h1>Python Programming Guide</h1>` |
| **First 10%** | 1 time | First paragraph contains keyword |
| **H2 Headings** | 2-4 times | `<h2>Python Programming Basics</h2>` |
| **H3 Headings** | 1-2 times | `<h3>Python Variables Explained</h3>` |
| **Meta Title** | 1 time | "Python Programming: Complete Guide 2025" |
| **Meta Description** | 1 time | "Learn Python programming from basics..." |
| **Content Body** | 1-2% density | ~25-50 times in 2,500 words |

### Secondary Keyword Strategy

- Use in H3 headings
- Natural distribution throughout content
- Create keyword clusters around main topic
- LSI (Latent Semantic Indexing) keywords: 10+ variations

### Keyword Density Calculation

```
Keyword Density = (Keyword Count / Total Word Count) × 100

Example:
- Keyword: "python programming"
- Word Count: 2,500
- Target Count: 25-50 occurrences (1-2% density)
```

### AI Prompt for Keyword Placement

```
Place the primary keyword "python programming" naturally in:
1. H1 heading (main title)
2. First 10% of the article content
3. At least 2 H2 headings
4. 1-2 H3 headings
5. Meta title (50-60 chars)
6. Meta description (120-160 chars)
7. Throughout content body (1-2% density, ~25-50 times in 2,500 words)
```

## Readability Standards

### Flesch-Kincaid Grade Level

- **Target**: 8th-10th grade (8.0-10.0)
- **Ideal**: 7th-9th grade for broad audience
- **Avoid**: College-level (12+) or above

### Sentence Length

- **Average**: 15-20 words per sentence
- **Maximum**: 25 words (rarely exceed)
- **Minimum**: 5 words (avoid fragments)

### Paragraph Length

- **Maximum**: 120 words per paragraph
- **Recommended**: 3-4 sentences per paragraph
- **Ideal**: 75-100 words per paragraph

### Readability Formula

```
Flesch Reading Ease = 206.835 - (1.015 × ASL) - (84.6 × ASY)

Where:
- ASL = Average Sentence Length (words/sentence)
- ASY = Average Syllables per Word
```

### Score Interpretation

| Score | Grade Level | Difficulty |
|-------|-------------|------------|
| 90-100 | 0-6 | Very Easy |
| 60-70 | 7-9 | Standard |
| 30-50 | 10-12 | College |
| 0-30 | 13+ | College Graduate |

### AI Prompt for Readability

```
Write content at 8th-10th grade reading level:
- Average sentence length: 15-20 words
- Maximum paragraph length: 120 words (3-4 sentences)
- Use active voice
- Use simple, common words
- Break up complex ideas with shorter sentences
```

## Images & Media

### Image Count

- **Minimum**: 4 images per 2,500-word article
- **Recommended**: 1 image per 500-600 words
- **Maximum**: No hard limit, but relevance matters

### Image Optimization Checklist

- [ ] Descriptive file name (hyphenated, lowercase)
- [ ] Alt text describing image content
- [ ] Title attribute (optional, for additional context)
- [ ] Proper dimensions (1200x630px for OG)
- [ ] Compressed for web (under 100KB)
- [ ] Lazy loading enabled
- [ ] Relevant to content

### Alt Text Best Practices

**Good Alt Text:**
```html
<img src="python-variables.jpg" 
     alt="Python variable declaration showing x = 10 and name = 'John'">
```

**Bad Alt Text:**
```html
<!-- Too generic -->
<img src="image1.jpg" alt="image">

<!-- Too long -->
<img src="chart.jpg" alt="A complex line chart showing the quarterly revenue growth of our company over the past five years with multiple data series and trend lines">
```

### AI-Generated Alt Text Process

1. Upload image to WordPress/Media Library
2. Click "Generate Alt" button
3. Review generated alt text for accuracy
4. Edit if necessary for better context
5. Save and publish

### Open Graph Image Requirements

- **Dimensions**: 1200 × 630 pixels
- **Aspect Ratio**: 1.91:1
- **File Size**: Under 5MB
- **Format**: JPEG, PNG, or GIF
- **Text**: Large, readable, brand logo optional

## Links

### Internal Linking

- **Quantity**: 3-5 internal links per 1,000 words
- **For 2,500 words**: 8-12 internal links
- **Anchor Text**: Descriptive, keyword-rich, natural

**Example:**
```html
<p>For more on Python best practices, see our 
<a href="/python-best-practices">Python Best Practices Guide</a>.</p>
```

### External Linking

- **Quantity**: 2-3 authoritative external links
- **Sources**: .gov, .edu, .org, established publications
- **Purpose**: Cite sources, provide evidence, add credibility

**Example:**
```html
<p>According to <a href="https://python.org">Python.org</a>, 
the language was created in 1991.</p>
```

### Link Placement Strategy

1. **Early in Content**: 1-2 internal links in first 300 words
2. **Throughout**: Natural link placement in relevant paragraphs
3. **End of Sections**: Link to related content
4. **Conclusion**: Internal links to action items

## Meta Tags

### SEO Title

- **Length**: 50-60 characters (max 70)
- **Keyword**: Include primary keyword near beginning
- **Format**: "Primary Keyword: Secondary Info | Brand Name"
- **Capitalization**: Title Case

**Example:**
```
Python Programming Guide: Complete Tutorial for Beginners 2025
```

### Meta Description

- **Length**: 120-160 characters (max 160)
- **Keyword**: Include primary keyword (near beginning)
- **Action**: Include compelling call-to-action
- **Uniqueness**: Unique per page

**Example:**
```
Learn Python programming from basics to advanced concepts. This comprehensive guide covers syntax, data structures, and practical examples for beginners.
```

### Open Graph Tags

```html
<meta property="og:title" content="Article Title (50-60 chars)">
<meta property="og:description" content="Meta description (120-160 chars)">
<meta property="og:image" content="https://example.com/og-image.jpg">
<meta property="og:image:width" content="1200">
<meta property="og:image:height" content="630">
<meta property="og:type" content="article">
<meta property="og:url" content="https://example.com/article">
```

### Twitter Card Tags

```html
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="Article Title">
<meta name="twitter:description" content="Meta description">
<meta name="twitter:image" content="https://example.com/twitter-image.jpg">
```

## Schema Markup

### Article/BlogPosting JSON-LD

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": "Article Title",
  "description": "Meta description",
  "author": {
    "@type": "Person",
    "name": "Author Name"
  },
  "publisher": {
    "@type": "Organization",
    "name": "Site Name",
    "logo": {
      "@type": "ImageObject",
      "url": "https://example.com/logo.png"
    }
  },
  "datePublished": "2025-01-01",
  "dateModified": "2025-01-02",
  "image": "https://example.com/featured-image.jpg",
  "keywords": "python, programming, tutorial, beginners"
}
</script>
```

### BreadcrumbList JSON-LD

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "Home",
      "item": "https://example.com/"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "Blog",
      "item": "https://example.com/blog/"
    },
    {
      "@type": "ListItem",
      "position": 3,
      "name": "Article Title",
      "item": "https://example.com/article-url/"
    }
  ]
}
</script>
```

### FAQ Schema (Deprecated - May 2026)

Note: Google deprecated FAQ rich results in May 2026. While FAQ schema markup still works, it no longer appears as rich results in Google Search. Focus on Article and Breadcrumb schemas for rich result eligibility.

## Common Mistakes to Avoid

1. **Keyword Stuffing**: Overusing keywords (density > 3%)
2. **Duplicate Content**: Copying from other sources
3. **Missing H1**: No main title or multiple H1 tags
4. **Skipping Heading Levels**: H1 → H3 without H2
5. **Thin Content**: Under 600 words for SEO tests
6. **Long Paragraphs**: Over 120 words per paragraph
7. **Missing Alt Text**: Images without descriptions
8. **Duplicate Meta Tags**: Same meta across pages
9. **Broken Links**: Internal or external links that don't work
10. **No Schema Markup**: Missing structured data
